About The Project

Our group will look at Census Income Data courtesy of UCI's Machine Learning Repository. Through roughly 15 input factors, we would like to train a machine learning model to be able to accurately predict whether a person's income is greater than, or lower than $50K.

Per the project requirements we will be using a number of different topics that we discussed in class. We will use scikit-learn as well as python, html/css/bootstrap, and javascript. We will additionally have to do a small amount of cleanup to our dataset in a jupyter notebook. We will create a new “cleaned csv” for analysis in this same notebook.

To accomplish this, we will test a number of different machine learning models. Those models are as follows:

  • Logistic Regression
  • K Nearest Neighbors
  • Random Forest
  • Neural Network

In an effort to dive deeper into our data, after looking at each of these models, we will turn our focus to the input factors. We will use a Correlation Matrix to analyze the strengths of the inputs and see if we can determine which ones are having more of an effect on the outcome. This will not affect which model is the best performer, but is an extra path our group wanted to explore for this project as we feel it could lead to more interesting insights about the data that we are working with.

Our process and analysis of which model performed best and why will be documented on a webpage. Here we plan to give a project overview, as well as detailed descriptions of how and why certain models and parameters were chosen, and how they affect the overall accuracy of the learning models. This will allow us to select our chosen model.

In addition to our analysis, we would like to create a webpage that will allow users to interact with the model that we find to be the most accurate so that they can enter in any input factors they like and get a prediction live from our model. To do this we will be using a flask server and hosting our website on Heroku. We will have to pickle our model and preprocessing functions. We are interested in pursuing this addition as it will allow us to not only develop a machine learning model, but to prove its applicability in creating a live demo of our work that anyone can participate in!
